# Contracts Public Notes

## Scope

This file is intentionally brief. Public documentation should describe the
currently published contract model and verification process, not serve as an
internal engineering journal.

## Current public position

- legacy marketplace contracts may remain live during staged migrations
- the new primary-market rollout uses an upgradeable proxy model
- collection contracts remain non-upgradeable in the current public design
- deployment and verification artifacts are expected to be published per release

## Expected public artifacts per release

- tagged source revision
- address book
- verification manifest
- ABI outputs
- any release-specific deployment notes needed for public verification

## Internal tracking

Detailed implementation status, engineering decisions in progress, and internal
next steps should be tracked outside this public repository.
